### Flowcharts

^9a8512

> Flowcharts are graphical representations of a process or algorithm, using different shapes & arrows to illustrate the sequence of steps. In scripting, flowcharts helps us visually plan & organise the logic of our scripts, making it easier to understand & communicate complex ideas.

^c29e0c

#### Benefits of Flowcharts
- **Clarity:** Flowcharts provide a clear overview of the logical flow, making it easier to grasp the script's structure
- **Communication:** Flowcharts serve as a visual aid when discussing script logic with team members or stakeholders.
- **Planning:** Before writing code, flowcharts allow us to plan & design our script's structure, reducing errors later
![[GAD170 - Flowchart Example.png]]
#### Flowchart Symbols
- **Start / End:** Represents the beginning or end of a process. Usually depicted as an oval shape.
- **Process / Action:** Represents a task or action to be performed. Shown as a rectangle.
- **Decision:** Represents a decision point, often involving conditionals like 'if-else' statements. Displayed as a Diamond
- **Connectors:** Used to connect different parts of a flowchart that are on different pages or sheets

![[GAD170 - Flowchart Example 2.png]]

### Pseudo Code

> Pseudo code is a simplified, human-readable representation of a program's logic. 
> It uses plain language to outline the sequence of steps in an algorithm.
> 
> In scripting, pseudo code helps us plan & communicate our code's structure before diving into actual coding

#### Advantages of Pseudo Code
- **Simplicity:** - Pseudo code eliminates the complexity of programming syntax, allowing us to focus on logic
- **Communication:** - Pseudo code is easy for team members & stakeholders to understand, even if they're not programmers
- **Design:** - Pseudo code helps in designing & refining the logical flow of a script before implementation

#### How to Create Pseudo Code 
- **Understand the Logic:** - Before writing actual code, grasp the overall sequence of actions your script needs to perform (Flowchart might help)
- **Start with Algorithm Start:** - Begin with a *'Program Start'* statement, marking the beginning of your Pseudo Code
- **Write Steps in Plain Language:** - Describe each step in plain language. Use words like *'Initialise, check, loop'* to convey actions
- **Use Variables & Conditions:** - Use variables & conditions (if-else statements) is Pseudo Code to represent logic
- **Loop & Iterations:** - Include Loops (for, while) in your pseudo code when necessary
- **Indicate the End of your  Program:** - End your Pseudo Code with a *'Program End'* statement, marking the conclusion of the program

### Deconstructing a Script
#### General Syntax 
- Class & Function Naming: ThisIsAnExampleOfAClass - 1st letter of each word is Upper-Case, Called *Pascal Case*
- Variable Naming: thisIsAnExampleOfAVariable - 1st letter of the 1st word is Lower-Case & each word after is Upper-Case, this is called *camelCase*
- ```;``` - is used to denote a *'full stop'* or *'go to next instruction'* we put this at the end of our lines of code
- ```()``` - Used for order of operations, as well as used for arguments for functions
- ```{}``` - Used to define scope of your program
- ```[]``` - Used to access elements of a collection
- ```.``` - Used to access data/functions within another class

> [!info] Remember: Your Code runs sequentially, it runs one line at a time. 

### Comments
Can be used to leave notes to your future self and other programmers, but also used to plan out your code. 
All of these are ignored by compilers so only we can see them!

```// Single Line Comment``` -  Allows for a Single Line Comment
```/* Milti-Line Comment*/``` - Allows us to write multiple lines, useful for longer messages 
```/// Summary``` - Allows us to summarise what functions & classes do
```#region Some RegionName``` 
```#end region```

#### Single Line Comment
- Single-line comments are annotations added to code for explanatory or documentation purposes. They can also be used for pseudo code to plan out your program.
- They are ignored by the compiler or interpreter and do not affect program execution. Single-line comments are great for adding brief explanations to code.
- They help developers understand complex logic, variables, or functions. Useful for leaving notes, marking temporary code, or disabling parts during debugging.
#### Multi-Line Comment
- Multi-line comments, also known as block comments, allow programmers to add longer explanations or notes to code. They span multiple lines and are ignored by the compiler or interpreter.
- Use multi-line comments for documenting functions, classes, modules, or providing detailed explanations. Valuable for describing algorithm steps, complex logic, and usage examples.
#### Summary
- C# also supports XML documentation comments for generating code documentation. They provide structured information about code elements and can be used by tools to generate documentation.
- Use summaries to describe functions, classes, and properties for enhanced clarity. This can be read for by your IDE and display the summary when hovering the name of function, class or property.
#### Regions
- Regions group related code together for improved readability. Ideal for separating code sections, methods, or properties. Ignored by the compiler and doesn't affect code behaviour.
- Regions in C# help maintain a neat and structured codebase, aiding development and maintenance. These regions can then be collapsed down in your IDE.

### Variables
- Variables are named containers used to store data values in programming. 
- They hold information that can be used and manipulated by the program. 
- Variables are declared by specifying the access type, a name and a data type. 

*Examples:* 
- ```public int score; private float temperature; public string playerName;```

![[GAD170 - Asset 1.png]]

#### Variable Access Types 
- **public** - Accessible from anywhere
- **private** - Accessible only within the same class
- **protected** - Accessible within the same class and derived classes
#### Variable Data Types
- **Int** - used for whole numbers i.e. 1,2,3,4.
- **Float** - used for decimals numbers i.e. 10.56f, when setting be sure to add an f after it; this lets unity know it's a float!
- **Bool** - used for true or false statements, i.e. public bool isGameOver = false;
- **String** - used for storing words, i.e. "Nathan is awesome!". The "" is used to let unity know it's a string.
- **Double** - used for more precision, can handle double the floating point numbers than a float.
- **Char** - used to store single characters like initials i.e. private char m_firstinitial = 'N'. The " is used to let unity know it's a char.
#### Variable Naming & Assignment
- Use descriptive names for variables (e.g., playerScore instead of ps).
- Follow a consistent naming convention for readability.
- Assign values to variables using the assignment operator =

*Example:*
- ```score = 100;```
- ```temperature = 23.5f;```
- ```playerName = "Alice";```
