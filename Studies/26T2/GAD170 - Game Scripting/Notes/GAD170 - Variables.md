# Notes

- think of Variables like *containers for data*
	- think of Cooking - having *'Jars'* for different ingredients, keeping the same kind of ingredients in the same jars
	- Keeping the same kind of data inside of the right containers
1. Always start with the Access Type
2. Define the type of data 
3. Define the name - Use camelCasing for the naming convention
	- Naming using camelCasing can help with quickly identifying wether it's a *variable* or a *function*
4. Assign a value / default value - using a '='
	- when using a single '=' - think of 'set' *ie:* ```public string myFirstName = "Nathan"``` or myFirstName is *set* to "Nathan"
- As programmers it's job to figure out what kind of data goes where, or what kind of data needs to be created
- Use descriptive name - you don't want to scroll to find a comment to figure out what the variable is 
	- Best to be able to figure out what it is by looking directly at it
	- Remember it needs to be able to be *easily* readable & understandable by anyone who reads your code 
- Can actually rename variables - right click & rename - not just re-type
#### Primitive Data Types
- Effectively exist in every language
- Include them at the top of the script
##### Strings
- Can only hold words or sentences
##### Boolean 
- Standard to add 'is' at the front, like 'isAlive' - as you're asking wether or not something is true or false
