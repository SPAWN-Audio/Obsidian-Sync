## Claude Summaries
### Variable Casting

Imagine you have a toy that's labeled "toy" but you _know_ it's actually a car. **Casting** is just you saying _"I know what this actually is"_ and treating it as such.

In Unity, variables have types (`int`, `float`, `string`, etc.). Sometimes Unity gives you something generic (like a `Component`) but you know it's actually something specific (like a `Rigidbody`). Casting lets you tell the compiler that:

```csharp
// Unity gives you a generic Component, but you know it's a Rigidbody
Component myThing = GetComponent<Component>();
Rigidbody rb = (Rigidbody)myThing; // <-- this is the cast
```

If you get it wrong and it's _not_ actually that type, your game will throw an error — so only cast when you're confident.

---
### Functions

A **function** is just a named set of instructions you can run whenever you want. Think of it like a recipe — you write the steps once, and you can "cook" it as many times as you like just by calling its name.

```csharp
void MakeExplosion()
{
    // all the explosion stuff goes in here
    PlaySound();
    SpawnParticles();
    ShakeCamera();
}

// Somewhere else in your code, just call it:
MakeExplosion();
```

`void` means the function does its job and walks away — it doesn't hand anything back to you.

---

### Functions with Parameters

Same as a regular function, but now you can **pass information in** when you call it. The recipe now accepts ingredients.

```csharp
void DealDamage(int amount)
{
    playerHealth -= amount;
}

// Now you can reuse it with different values:
DealDamage(10);
DealDamage(50);
```

Parameters sit inside the parentheses. You can have as many as you need, separated by commas.

---

### Return Functions

A function that **gives something back** to whoever called it. Instead of `void`, you declare what _type_ of thing it will return, and you use the `return` keyword to hand it over.

```csharp
int AddNumbers(int a, int b)
{
    return a + b;  // hands the result back
}

// Now you can catch what it returns:
int result = AddNumbers(3, 5); // result is now 8
```

Think of it like asking someone a question — a `void` function is someone who just does a task silently, a `return` function is someone who does the task _and tells you the answer_.

---

### UI Buttons

A **UI Button** is the clickable button element from Unity's UI system. You create it in the scene (via _GameObject → UI → Button_), and then wire up a function to run when it's clicked.

You can do this two ways:

**In the Inspector** — drag your script onto the button's `OnClick()` event and pick the function.

**In code** — grab the button and add a listener:

```csharp
using UnityEngine.UI;

public Button myButton;

void Start()
{
    myButton.onClick.AddListener(WhenButtonClicked);
}

void WhenButtonClicked()
{
    Debug.Log("Button was clicked!");
}
```

The `OnClick` event is just Unity saying _"when someone clicks this, run these functions."_

---

### UI Text

A **UI Text** component displays a string of text on screen (like a score, health counter, dialogue, etc.). The modern Unity way uses **TextMeshPro** (`TMP_Text`), but the concept is the same.

```csharp
using TMPro;

public TMP_Text scoreText;

void UpdateScore(int newScore)
{
    scoreText.text = "Score: " + newScore;
}
```

You create the text element in the scene (_GameObject → UI → Text - TextMeshPro_), drag it into your script's public variable slot in the Inspector, and then just set `.text` to whatever string you want displayed. It updates live in the game.

## Variable Casting
Variable casting involves converting a value of one data type into another data type. This is particularly useful when you need to perform operations or assignments between different types of variables.

- Implicit Casting: Automatically performed by the compiler when there's no data loss (e.g., int to float).
- Explicit Casting (Type Casting); Manually performed by the programmer when data loss is possible (e.g., float to int).

![[GAD170 - Variable Casting 1.png]]

### Using Variable Casting
In these examples we are doing implicit casting as we can store a 5 into a float.
 
 However to go the other way we need to explicitly tell the compiler how to handle this extra data. In this case, we cut off any decimal number so the result will be 7.

A good example of where you might want to do this, is imagining if you have your health stored as a float, you probably don't want to display 76.57f on the user interface in your game. Instead keeping it as a float on the backend for calculations, but display it as a whole number to the user.

## Functions with Parameters
Functions in Unity C# can accept parameters, allowing you to pass values into the function to be used in its execution.
### Benefits:
- Customisation: Pass specific values to adapt the behaviour of the function.
- Reusability: Reuse the same function for different scenarios by changing parameter values.
- Modularity: Divide complex tasks into smaller functions that interact through parameters.

A good indication for when you should use a function with parameters is when you are duplicating a function and changing values; these values could be parameters. You can also have as many parameters as you like, and they can be of any type.

## Return Functions
Functions in Unity C# can return values, which allows them to provide computed results or data back to the caller (where it was called).
### Benefits:
- Data Extraction: Get computed or processed data from a function.
- Reusability: Use the same function in different contexts with varying results.
- Expression Evaluation: Return results of mathematical or logical computations.

A good indication when you should use a return function is when you are setting global variables within it. Instead return the value and set the variable to it.

## UI Text
We can use text to communicates information to players, provides instructions, feedback, and narrative elements or enhance the user experience and engagement.

There are two options available in Unity, the default Text Component: basic text rendering. Or you could use TextMesh Pro which has advanced text rendering with rich features.

Whilst Unity's default is good, it is recommended to use TextMesh Pro given the following features:
- Rich text support: Bold, italics, colors, and more.
- Text styling options for better visual appeal.
- Compatibility with different platforms and resolutions.
