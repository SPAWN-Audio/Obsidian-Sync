## Claude Code Summaries
### Materials
A **Material** is what determines how the surface of a 3D object _looks_ — its colour, texture, shininess, transparency, and so on. Think of it like paint and finish on a car. The car shape is the mesh; the material is what makes it look red and glossy vs. rusty and matte.

You create Materials in the Project window (_right-click → Create → Material_), tweak them in the Inspector, and drag them onto GameObjects.

From code, you can read and change a material's properties at runtime:

```csharp
public Renderer myRenderer;

void MakeItRed()
{
    // .material gives you THIS object's own copy of the material
    myRenderer.material.color = Color.red;
}
```

> ⚠️ **Important — `.material` vs `.sharedMaterial`:**

```csharp
// .material — creates a unique copy just for this object (safe to edit, uses more memory)
myRenderer.material.color = Color.red;

// .sharedMaterial — edits the original asset, changes ALL objects using it
myRenderer.sharedMaterial.color = Color.red;
```

Use `.material` when you want to change one object individually. Use `.sharedMaterial` only when you deliberately want to change every object that shares it.

**Swapping the whole material out entirely:**

```csharp
public Material damagedMaterial;

void TakeDamage()
{
    // Swap to a completely different material
    myRenderer.material = damagedMaterial;
}
```

You can also change other properties like transparency or texture:

```csharp
// Change opacity (material needs to support transparency)
Color c = myRenderer.material.color;
c.a = 0.5f; // 0 = invisible, 1 = fully visible
myRenderer.material.color = c;

// Swap the texture
myRenderer.material.mainTexture = myNewTexture;
```

### Animations

Unity's animation system lets objects move, rotate, scale, or change over time in pre-authored ways. There are two main pieces:

- **Animation Clip** — the actual recorded motion (like "Walk", "Jump", "Die")
- **Animator** — the controller that decides _which_ clip to play and _when_

Think of the Animator as a flowchart of states. Your character is always in one state (Idle, Walking, Jumping...) and conditions you set trigger the transitions between them. You set this up visually in the **Animator window**.

From code, you talk to the Animator by setting **parameters** — booleans, floats, ints, or triggers — that drive those transitions:

```csharp
public Animator animator;

void Move()
{
    // Set a float — e.g. blend between idle and walk based on speed
    animator.SetFloat("Speed", 5.0f);
}

void Jump()
{
    // Trigger a one-shot transition
    animator.SetTrigger("Jump");
}

void Die()
{
    // Set a bool
    animator.SetBool("IsDead", true);
}
```

The string names (`"Speed"`, `"Jump"`) must match **exactly** what you named the parameters in the Animator window.

**Playing an animation state directly by name:**

```csharp
animator.Play("Run");
```

**Checking what's currently playing:**

```csharp
// Get info about the current animation state
AnimatorStateInfo stateInfo = animator.GetCurrentAnimatorStateInfo(0);

if (stateInfo.IsName("Jump"))
{
    Debug.Log("Currently jumping!");
}
```

> 💡 **Tip:** The `0` in `GetCurrentAnimatorStateInfo(0)` refers to the **layer** — most setups just use layer 0. Layers let you run separate animations on different parts of a character at once (e.g. legs running while arms shoot).

### Particles

A **Particle System** is a component that spits out lots of small sprites or meshes rapidly — used for fire, smoke, sparks, explosions, rain, magic effects, basically anything fluffy or chaotic that would be painful to animate by hand.

You add one via _GameObject → Effects → Particle System_. Almost everything about it (speed, size, colour, lifetime, emission rate) is configured in the Inspector — and most of the time you barely need to touch it from code.

The most common thing to do in code is **play and stop** it:

```csharp
public ParticleSystem explosionEffect;

void Explode()
{
    explosionEffect.Play();
}

void StopSmoke()
{
    smokeEffect.Stop();
}
```

**Check if it's currently running:**

```csharp
if (explosionEffect.isPlaying)
{
    Debug.Log("Still going!");
}
```

**One-shot effects** (explosion, impact spark) — the cleanest pattern is to Instantiate the particle Prefab, then let it destroy itself:

```csharp
public ParticleSystem impactPrefab;

void OnCollisionEnter(Collision col)
{
    ParticleSystem effect = Instantiate(impactPrefab, transform.position, Quaternion.identity);
    
    // Destroy it after its duration is up
    Destroy(effect.gameObject, effect.main.duration);
}
```

**Changing particle properties from code:**

```csharp
// You have to grab the module first, then edit it
var main = myParticles.main;
main.startColor = Color.blue;
main.startSpeed = 10f;
main.startSize = 0.5f;
```

> ⚠️ **Note:** Particle modules (like `main`, `emission`, `colorOverLifetime`) look like they return a copy but they actually directly modify the system — no need to assign them back. This is a quirky Unity design, just roll with it.

**Controlling emission rate at runtime:**

```csharp
var emission = myParticles.emission;
emission.rateOverTime = 50; // 50 particles per second
```

## Quick Cheat Sheet:

| Concept             | One-liner                                                           |
| ------------------- | ------------------------------------------------------------------- |
| Material            | The paint and finish on a 3D object's surface                       |
| `.material`         | Edit this one object's look only                                    |
| `.sharedMaterial`   | Edit all objects using that material at once                        |
| Animation Clip      | A recorded motion (Walk, Jump, Die)                                 |
| Animator            | The flowchart that decides which clip plays when                    |
| Animator Parameters | The values your code sets to trigger transitions                    |
| Particle System     | A component that rapidly spawns small effects (fire, smoke, sparks) |
| `Play()` / `Stop()` | Start or stop a particle system from code                           |

## Materials
Materials define the appearance of objects in Unity.They control how light interacts with surfaces to create realistic visuals.
### Elements of a material:
#### Shader:
- Determines how light interacts with the surface.
- Defines attributes like colour, transparency, and specular highlights.
#### Textures:
- Images used to add detail and realism to a material.
- Types include albedo, normal maps, specular maps, and more.
#### Properties: 
- Adjust settings specific to the shader, such as glossiness or emission.
### To Create one
Use the Asset menu to create a new material. Assign shaders and adjust properties in the Inspector.

## Animations
Animations bring life and movement to your game's characters and objects. Unity's Animation system allows you to create dynamic and interactive animations.
### Benefits:
- Visual Appeal: Make characters and objects move in response to actions.
- Gameplay Feedback: Provide feedback to players through animations.
- Storytelling: Enhance storytelling with cinematic and narrative animations.
### Two Major Concepts:
#### Animator Controller:
- Controls the flow of animations and transitions between states.
- Connects animation clips, states, and transitions.
#### Animation Clips:
- Individual animation sequences for different actions.
- Can include keyframes, poses, and curves.

Keep in mind this is a big topic, I recommend reviewing the [Unity Manual](https://docs.unity3d.com/Manual/AnimationSection.html)

## Particles
Particles are small visual elements that simulate natural phenomena like fire, smoke, and explosions. 

Unity's Particle System enables the creation of dynamic and immersive visual effects. Again another big topic, I recommend referring to the [Unity Manual](https://docs.unity3d.com/Manual/AnimationSection.html)

### Key Elements
#### Particle System
- Core component to create and control particle effects.
- Simulates particles' behaviour over time.
#### Modules
- Emit: Set initial position, velocity, and shape of particles.
- Shape: Define where particles are emitted.
- Lifetime: Control the lifespan of particles.
- Renderer: Define how particles are displayed.
#### Textures & Materials
- Apply textures and materials to particles for various effects.

![[GAD170 - Particle 2.png|592x365]]

### Particle Examples
