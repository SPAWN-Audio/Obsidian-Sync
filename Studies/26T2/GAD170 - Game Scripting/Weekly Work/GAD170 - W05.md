## Claude Code Summaries
### Turning Objects On/Off

Every GameObject in Unity has an **active state** — it's either on or off. When an object is turned off, it completely disappears from the game — invisible, non-collidable, and all its scripts stop running. Turning it back on brings it back exactly as it was.

You control this with `SetActive()`:

```csharp
// Turn the object off
gameObject.SetActive(false);

// Turn it back on
gameObject.SetActive(true);
```

You can also do it to _other_ objects, not just the one the script is on:

```csharp
public GameObject shield;

void ActivateShield()
{
    shield.SetActive(true);
}

void DeactivateShield()
{
    shield.SetActive(false);
}
```

And you can **check** whether something is currently active:

```csharp
if (shield.activeSelf)
{
    Debug.Log("Shield is on!");
}
```

> 💡 **Tip:** Turning objects off and on is much cheaper than destroying and recreating them constantly. For things that appear and disappear frequently (like bullets or enemies), consider toggling instead of destroying — or better yet, use a **Object Pool** pattern. Speaking of creating objects though...

### Prefabs

A **Prefab** is a saved, reusable template of a GameObject — with all its components, settings, and child objects baked in.

Think of it like a stamp. You design the stamp once (your enemy, your bullet, your pickup item), save it as a Prefab, and then you can stamp out as many identical copies as you want — at any time, from anywhere.

You create a Prefab by simply **dragging a GameObject from your scene into the Project window**. Unity saves it as a `.prefab` file. From then on, that file _is_ the master template.

The real power: if you change the Prefab, **all instances in your game update automatically.** Change the enemy's colour in the Prefab — every single enemy in every scene updates. No need to change them one by one.

```csharp
// Reference to the Prefab asset
public GameObject enemyPrefab;
```

You drag the Prefab file (from the Project window, not the scene) into that slot in the Inspector. Now your script knows about it and can use it to stamp out copies — which brings us to...

---
### Instantiate

**Instantiate** is the function that actually stamps out a copy of a Prefab (or any GameObject) and places it live into your running game. This is how bullets get fired, enemies get spawned, and collectables get dropped.

```csharp
public GameObject bulletPrefab;

void Shoot()
{
    Instantiate(bulletPrefab);
}
```

That works, but the bullet would appear at the world origin `(0, 0, 0)`. You almost always want to control **where** it appears and **which way** it's facing:

```csharp
void Shoot()
{
    // Spawn at this object's position, with this object's rotation
    Instantiate(bulletPrefab, transform.position, transform.rotation);
}
```

You can also **catch a reference** to the thing you just spawned, so you can do something with it immediately:

```csharp
void SpawnEnemy()
{
    GameObject newEnemy = Instantiate(enemyPrefab, spawnPoint.position, Quaternion.identity);
    
    // Now you can talk to it straight away
    newEnemy.GetComponent<Enemy>().health = 200;
}
```

`Quaternion.identity` just means "no rotation — spawn it facing the default direction."

> ⚠️ **Watch out:** Spawning lots of objects with Instantiate every frame gets expensive fast. For high-frequency spawning (bullets, particles), look into **Object Pooling** — but Instantiate is perfectly fine for most normal use.

---
### Destroy

**Destroy** removes a GameObject from the game entirely. It's gone — from the scene, from memory, from existence. This is how enemies die, bullets disappear, and pickups vanish when collected.

```csharp
// Destroy this object immediately
Destroy(gameObject);

// Destroy a different object
Destroy(enemyObject);

// Destroy after a delay (in seconds) — great for explosions, effects
Destroy(gameObject, 3f);
```

You can also use Destroy to remove just a **component** from an object, leaving the object itself alive:

```csharp
// Removes just the Rigidbody component, keeps the GameObject
Destroy(GetComponent<Rigidbody>());
```

A very common pattern is a bullet destroying itself after a few seconds so it doesn't fly off forever and eat memory:

```csharp
public class Bullet : MonoBehaviour
{
    void Start()
    {
        Destroy(gameObject, 5f); // auto-cleanup after 5 seconds
    }

    void OnCollisionEnter(Collision col)
    {
        Destroy(gameObject); // also destroy on impact
    }
}
```

> ⚠️ **Important:** After calling `Destroy()`, the object isn't gone _instantly_ — it finishes the current frame first and gets cleaned up at the end. This means you can still accidentally reference it within the same frame. To check if something has been destroyed, use your null check — destroyed objects evaluate as null.

### Quick Cheat Sheet:

|Concept|One-liner|
|---|---|
|`SetActive(false/true)`|Turn an object off or on without destroying it|
|Prefab|A saved reusable template for a GameObject|
|`Instantiate()`|Stamp a live copy of a Prefab into the running game|
|`Destroy()`|Permanently remove a GameObject (or component) from the game|

## Prefabs
A prefab is a reusable template for creating GameObjects. Prefabs streamline the process of object creation and management.
### Benefits:
- Consistency: Ensure consistent properties across instances.
- Efficiency: Create and modify objects in one place.
- Easy Updates: Changes to a prefab affect all instances.
### Creation:
- Design a GameObject in the scene.
- Drag it from the Hierarchy to the Assets as a prefab.
## Instantiate
Instantiation is the process of creating new instances (objects) in Unity.
It's a fundamental concept for dynamically adding and managing elements in a scene.
This allows us to create instances of object or more commonly instances of prefabs of objects in our
scenes.

### Instantiate Example
This script manages the player's health and visual effects related to taking damage and dying. When the player takes damage, the OnPlayerDamaged function is called.

It then creates a visual effect (like a blood splatter or damage indicator) at the player's position to show that damage has been taken.

If the player's health drops to zero or below, the Die method is called. This method spawns a death effect (such as an explosion or fading animation) at the player's position and automatically destroys it after 2 seconds to keep the scene clean.

## Destroy
In Unity, "destroying" an object means removing it from the scene.
Destroying objects is crucial for memory management and game mechanics.
So we can either destroy the `gameobject` immediately using destroy. Alternatively we can pass in a float of seconds to delay the destroy.
